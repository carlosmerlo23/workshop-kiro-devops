## Objetivos del Workshop

Al finalizar este workshop, los participantes podran:

1. Comprender como Kiro acelera las practicas DevOps en el ciclo de desarrollo
2. Configurar Kiro como herramienta central para proyectos DevOps
3. Generar pipelines de CI/CD usando prompts inteligentes
4. Crear pruebas automatizadas y documentacion de pipelines con asistencia de IA
5. (Dia 2) Generar plantillas de Infrastructure as Code con Kiro
6. (Dia 2) Implementar monitoreo basico y documentacion operativa

## Prerequisitos

### Para los participantes:
- Cuenta de GitHub activa
- Conocimientos basicos de Git (clone, push, pull, branches)
- Familiaridad con algun lenguaje de programacion (Python, JavaScript, Java)
- Laptop con permisos de administrador

### Software necesario:
- [Kiro IDE](https://kiro.dev) instalado y configurado
- Git CLI instalado
- Node.js 18+ o Python 3.10+ (segun stack del equipo)
- Cuenta de GitHub con acceso a GitHub Actions
- Terminal/consola funcional

### Preparacion previa (enviar antes del 6 de julio):
- Instalar Kiro IDE desde https://kiro.dev
- Crear un repositorio de prueba en GitHub
- Verificar que pueden hacer push a su repositorio

## Metodologia

Este workshop sigue un enfoque **learn-by-doing**:

1. **Teoria breve** (20%): Conceptos clave explicados de forma concisa
2. **Demostracion en vivo** (20%): El facilitador muestra el flujo completo
3. **Practica guiada** (50%): Los participantes replican y experimentan
4. **Discusion y review** (10%): Revision de resultados en grupo

### Flujo de cada Lab:
```
Contexto → Demo del facilitador → Ejercicio guiado → Ejercicio libre → Review
```
